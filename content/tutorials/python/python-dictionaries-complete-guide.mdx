# Python Dictionaries: The Complete Guide

If lists are Python's workhorse, **dictionaries** are its Swiss Army knife. They let you store data as meaningful **key-value pairs** instead of relying on numeric positions — making your code more readable and your lookups lightning fast. This comprehensive guide walks through creation, access patterns, mutation, and every built-in dictionary method.

<YouTubeEmbed id="dQw4w9WgXcQ" title="Python Dictionaries Complete Guide" />

## Key Takeaways

- Dictionaries store data as **key:value pairs** inside curly braces `{}`.
- Keys must be **hashable** (strings, numbers, tuples) — lists and dicts cannot be keys.
- As of Python 3.7+, dictionaries **maintain insertion order**.
- Python provides **11 built-in methods** for powerful dictionary manipulation.
- Use `.get()` for safe access that avoids `KeyError` exceptions.

## What Is a Dictionary?

A **dictionary** is a mutable, ordered (Python 3.7+) collection of key-value pairs. Each key maps to exactly one value, and duplicate keys are not allowed — assigning to an existing key overwrites its value.

```python
# A simple dictionary
student = {
    "name": "Priya",
    "age": 22,
    "major": "Computer Science"
}

print(student)
# {'name': 'Priya', 'age': 22, 'major': 'Computer Science'}
```

## Creating Dictionaries

### Literal Syntax

```python
config = {
    "host": "localhost",
    "port": 5432,
    "debug": True
}
```

### The `dict()` Constructor

```python
# From keyword arguments
config = dict(host="localhost", port=5432, debug=True)

# From a list of tuples
pairs = [("host", "localhost"), ("port", 5432)]
config = dict(pairs)
```

## Key Restrictions: What Can Be a Key?

Keys must be **hashable** — meaning they need a consistent hash value. Immutable types qualify; mutable types do not.

```python
# Valid keys
valid = {
    "name": "Alice",       # string
    42: "answer",           # integer
    (1, 2): "coordinates",  # tuple (if all elements are hashable)
    3.14: "pi"              # float
}

# Invalid keys — these raise TypeError
# {[1, 2]: "nope"}       # list is mutable → not hashable
# {{"a": 1}: "nope"}     # dict is mutable → not hashable
```

## Accessing Values

Dictionaries use **keys**, not indices, for access. Keys are **case-sensitive**.

```python
student = {"Name": "Aarav", "Age": 21, "Grade": "A"}

print(student["Name"])   # 'Aarav'
print(student["Grade"])  # 'A'

# student["name"] → KeyError (lowercase 'n' ≠ uppercase 'N')
```

### Integer Keys for Index-Like Access

You can use integers as keys to mimic index-based access, though this is a stylistic choice rather than true indexing.

```python
ranking = {1: "Gold", 2: "Silver", 3: "Bronze"}
print(ranking[1])  # 'Gold' — this is key lookup, not index lookup
```

## Adding and Updating Entries

```python
profile = {"name": "Meera", "age": 25}

# Add a new key-value pair
profile["city"] = "Mumbai"

# Update an existing value
profile["age"] = 26

print(profile)
# {'name': 'Meera', 'age': 26, 'city': 'Mumbai'}
```

## All 11 Dictionary Methods

### 1. `.clear()` — Remove All Entries

```python
data = {"a": 1, "b": 2}
data.clear()
print(data)  # {}
```

### 2. `.copy()` — Shallow Copy

```python
original = {"x": 10, "y": 20}
clone = original.copy()
clone["x"] = 99
print(original["x"])  # 10 — original is unaffected
```

### 3. `.fromkeys(keys, value)` — Create from Key Sequence

```python
subjects = ["math", "science", "english"]
scores = dict.fromkeys(subjects, 0)
print(scores)  # {'math': 0, 'science': 0, 'english': 0}
```

### 4. `.get(key, default)` — Safe Access

The safest way to retrieve a value — returns `default` instead of raising `KeyError`.

```python
settings = {"theme": "dark", "language": "en"}

print(settings.get("theme"))          # 'dark'
print(settings.get("font_size", 14))  # 14 — key missing, default returned
print(settings.get("font_size"))      # None — default is None
```

### 5. `.items()` — Key-Value Pairs as View

Returns a **view object** of `(key, value)` tuples — ideal for iteration.

```python
menu = {"coffee": 4.50, "tea": 3.00, "juice": 5.25}

for item, price in menu.items():
    print(f"{item}: ${price:.2f}")

# Convert to list of tuples
pairs = list(menu.items())
print(pairs)  # [('coffee', 4.5), ('tea', 3.0), ('juice', 5.25)]
```

### 6. `.keys()` — All Keys as View

```python
menu = {"coffee": 4.50, "tea": 3.00, "juice": 5.25}
print(list(menu.keys()))  # ['coffee', 'tea', 'juice']
```

### 7. `.values()` — All Values as View

```python
menu = {"coffee": 4.50, "tea": 3.00, "juice": 5.25}
print(list(menu.values()))  # [4.5, 3.0, 5.25]
```

### 8. `.pop(key, default)` — Remove and Return

```python
inventory = {"apples": 5, "bananas": 3, "cherries": 12}

removed = inventory.pop("bananas")
print(removed)    # 3
print(inventory)  # {'apples': 5, 'cherries': 12}

# With a default to avoid KeyError
missing = inventory.pop("grapes", 0)
print(missing)    # 0
```

### 9. `.popitem()` — Remove Last Inserted Pair

```python
tasks = {"morning": "exercise", "afternoon": "code", "evening": "read"}

last = tasks.popitem()
print(last)   # ('evening', 'read')
print(tasks)  # {'morning': 'exercise', 'afternoon': 'code'}
```

### 10. `.setdefault(key, default)` — Get or Insert

Returns the value if the key exists; otherwise, **inserts** the key with the default value and returns it.

```python
preferences = {"theme": "dark"}

# Key exists — returns existing value, no change
theme = preferences.setdefault("theme", "light")
print(theme)  # 'dark'

# Key missing — inserts and returns default
font = preferences.setdefault("font_size", 16)
print(font)         # 16
print(preferences)  # {'theme': 'dark', 'font_size': 16}
```

### 11. `.update(other)` — Merge Another Dict

```python
defaults = {"color": "blue", "size": "medium", "quantity": 1}
overrides = {"size": "large", "gift_wrap": True}

defaults.update(overrides)
print(defaults)
# {'color': 'blue', 'size': 'large', 'quantity': 1, 'gift_wrap': True}
```

## Iterating Over Dictionaries

```python
scores = {"Alice": 92, "Bob": 85, "Carol": 97}

# Iterate over keys (default behavior)
for name in scores:
    print(name)

# Iterate with both key and value
for name, score in scores.items():
    print(f"{name} scored {score}")
```

## Nested Dictionaries

Dictionaries can contain other dictionaries, enabling rich, hierarchical data structures.

```python
university = {
    "CS101": {
        "title": "Intro to Programming",
        "instructor": "Dr. Sharma",
        "students": 120
    },
    "MA201": {
        "title": "Linear Algebra",
        "instructor": "Dr. Patel",
        "students": 85
    }
}

print(university["CS101"]["instructor"])  # 'Dr. Sharma'
```
