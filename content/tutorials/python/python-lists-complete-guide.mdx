# Python Lists: The Complete Guide

If there is one data structure every Python programmer uses daily, it is the **list**. Lists are the Swiss-army knife of Python — flexible enough to hold anything, powerful enough to model almost any sequential data, and backed by a rich set of built-in methods that keep your code concise. This article covers everything you need to go from zero to confident.

<YouTubeEmbed id="dQw4w9WgXcQ" title="Python Lists — Complete Guide" />

### Key Takeaways

- Lists are **ordered**, **mutable**, and **dynamic** collections enclosed in square brackets.
- Elements can be of any type — including other lists (nesting).
- Positive and negative indexing give you flexible access to elements.
- Methods like `append()`, `extend()`, `insert()`, `remove()`, `pop()`, and `sort()` modify a list in place.
- Built-in functions such as `len()`, `sum()`, `min()`, `max()`, and `list()` work seamlessly with lists.

---

## What Is a List?

A **list** is a sequential data structure that stores an ordered collection of items inside **square brackets**, separated by **commas**.

```python
fruits = ["apple", "banana", "cherry"]
scores = [95, 82, 74, 91]
mixed = [1, "hello", 3.14, True]
```

Imagine tracking the grades of 200 students. Creating `grade_1`, `grade_2`, … `grade_200` is impractical. A single list solves this:

```python
grades = [88, 76, 95, 64, 90]  # one variable, many values
```

## Core Characteristics of Lists

**Ordered** — items maintain the sequence in which they were added. **Arbitrary types** — a list can hold integers, strings, floats, booleans, and even other lists in the same collection. **Dynamic** — there is no fixed size; lists grow and shrink at runtime.

```python
everything = [42, "Python", 3.14, False, [1, 2, 3]]
```

### Index Access — Positive and Negative

Every element has a **positional index** starting at `0`. Python also supports **negative indexing**, where `-1` refers to the last element.

```python
colors = ["red", "green", "blue", "yellow"]

print(colors[0])   # "red"
print(colors[-1])  # "yellow"
print(colors[-2])  # "blue"
```

> 🖼️ **Visual Suggestion:** A numbered row of boxes showing both positive indices (0, 1, 2, 3) above and negative indices (-4, -3, -2, -1) below.

### Nested Lists

Lists can contain other lists, creating a matrix-like structure. Access inner elements with **chained indexing**.

```python
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9],
]

print(matrix[0])       # [1, 2, 3]   — first row
print(matrix[1][2])    # 6           — second row, third column
print(matrix[2][-1])   # 9           — last element of last row
```

### Mutable

Unlike strings, lists can be changed after creation. You can reassign individual elements directly.

```python
pets = ["cat", "dog", "fish"]
pets[1] = "hamster"
print(pets)  # ["cat", "hamster", "fish"]
```

## Essential List Methods

### Sorting with `sort()`

The `sort()` method orders elements **in place** (it does not return a new list). Pass `reverse=True` to sort in descending order.

```python
numbers = [5, 2, 9, 1, 7]
numbers.sort()
print(numbers)  # [1, 2, 5, 7, 9]

numbers.sort(reverse=True)
print(numbers)  # [9, 7, 5, 2, 1]
```

### `append()` vs `extend()`

`append()` adds **one item**; `extend()` unpacks an iterable and adds **each item** individually.

```python
langs = ["Python", "JavaScript"]
langs.append("Go")
print(langs)  # ["Python", "JavaScript", "Go"]

langs.extend(["Rust", "C++"])
print(langs)  # ["Python", "JavaScript", "Go", "Rust", "C++"]
```

### `index()` and `insert()`

`index()` returns the position of the **first occurrence**. `insert(position, value)` places an element at the given index, shifting items right.

```python
letters = ["a", "b", "c", "b", "d"]
print(letters.index("b"))  # 1

nums = [10, 30, 40]
nums.insert(1, 20)
print(nums)  # [10, 20, 30, 40]
```

### `remove()` vs `pop()`

`remove()` deletes the **first occurrence** of a value. `pop()` removes by index and **returns** the element — without an argument it removes the last item.

```python
colors = ["red", "green", "blue", "green"]
colors.remove("green")
print(colors)  # ["red", "blue", "green"]

stack = [1, 2, 3, 4, 5]
last = stack.pop()       # returns 5
second = stack.pop(1)    # returns 3
print(stack)             # [1, 4]
```

> 🖼️ **Visual Suggestion:** Flowchart — "Need to remove an element?" → by value → `remove()` / by position → `pop(index)` / from the end → `pop()`.

## Built-in Functions That Work with Lists

Python provides several built-in functions that accept lists as arguments.

```python
temps = [22.5, 18.0, 31.2, 25.7]

print(len(temps))   # 4
print(min(temps))   # 18.0
print(max(temps))   # 31.2
print(sum(temps))   # 97.4
```

### `list()` — Type Conversion

`list()` converts other iterables — strings, tuples, ranges — into lists.

```python
print(list("hello"))        # ["h", "e", "l", "l", "o"]
print(list(range(5)))       # [0, 1, 2, 3, 4]
print(list((10, 20, 30)))   # [10, 20, 30]
```

## Putting It All Together

```python
scores = [85, 92, 78, 95, 88, 73, 91]

scores.sort()
scores.append(100)
lowest = scores.pop(0)

print(f"Removed lowest: {lowest}")                   # 73
print(f"Count  : {len(scores)}")                     # 7
print(f"Average: {sum(scores) / len(scores):.1f}")   # 89.9
```

Lists are foundational to almost every Python program. Master them, and you will have a reliable tool for everything from quick scripts to large-scale applications.
