# The Complete Guide to Regular Expressions in Python

Regular expressions — **regex** for short — are one of the most powerful tools in any programmer's arsenal. They let you define **search patterns** that can match, extract, validate, and transform text with surgical precision. Instead of writing dozens of lines of string manipulation code, a single regex pattern can do the job.

This guide covers everything from basic pattern matching to advanced features like groups, backreferences, and flags — culminating in a practical password strength validator you can use in real projects.

## What Are Regular Expressions?

A regular expression is a **sequence of characters that defines a search pattern**. Think of it as a super-powered find-and-replace tool. Instead of searching for exact text like `"hello"`, you can search for **patterns** like "any email address" or "any phone number that starts with +91."

**Why use regex?**

- **Validate input** — emails, phone numbers, passwords, dates
- **Search and extract** — find specific patterns in large text
- **Replace and clean** — transform messy data efficiently
- **Parse text files** — extract structured information from unstructured text

## The `re` Module — Getting Started

Python's regex functionality lives in the built-in `re` module:

```python
import re
```

### `re.search()` — Find the First Match

```python
import re

text = "Hello, welcome to Python programming. Python is awesome!"

result = re.search("Python", text)
print(result)
# <re.Match object; span=(18, 24), match='Python'>
```

`search()` scans the entire string and returns the **first match** as a `Match` object. If no match exists, it returns `None`:

```python
result = re.search("Java", text)
print(result)  # → None
```

### `re.findall()` — Find All Matches

```python
result = re.findall("Python", text)
print(result)  # → ['Python', 'Python']
```

`findall()` returns a **list** of all non-overlapping matches.

### `re.match()` — Match at the Beginning Only

```python
text = "Python is awesome"

print(re.match("Python", text))   # → Match object (starts at position 0)
print(re.match("awesome", text))  # → None ("awesome" isn't at the start)
```

`match()` only checks if the pattern appears **at the very beginning** of the string. Use `search()` to scan the entire string.

## Special Characters — The Real Power of Regex

Literal text matching is useful, but regex becomes powerful with **metacharacters** — special characters that represent patterns rather than literal text.

### The Dot (`.`) — Match Any Character

```python
text = "cat, bat, hat, mat"

result = re.findall("c.t", text)
print(result)  # → ['cat']
```

The dot matches **any single character** (except newline), so `c.t` matches "cat", "cot", "c9t", etc.

### Square Brackets (`[]`) — Character Sets

```python
result = re.findall("[cbh]at", text)
print(result)  # → ['cat', 'bat', 'hat']
```

`[cbh]at` matches any string starting with `c`, `b`, or `h`, followed by `at`. The `m` in "mat" isn't in the set, so it's excluded.

### Quantifiers — Controlling Repetition

| Quantifier | Meaning | Example |
|-----------|---------|---------|
| `*` | Zero or more | `ca*t` matches "ct", "cat", "caat" |
| `+` | One or more | `ca+t` matches "cat", "caat" (not "ct") |
| `?` | Zero or one (optional) | `colou?r` matches "color" and "colour" |
| `{n}` | Exactly n times | `\d{3}` matches "123" |
| `{n,m}` | Between n and m times | `\d{1,2}` matches "1" or "12" |

```python
text = "ct, cat, caat, caaat"

print(re.findall("ca*t", text))   # → ['ct', 'cat', 'caat', 'caaat']
print(re.findall("ca+t", text))   # → ['cat', 'caat', 'caaat']
```

```python
text = "The color of the shirt and the colour of the wall"

print(re.findall("colou?r", text))  # → ['color', 'colour']
```

```python
text = "File1 File22 File333"

result = re.findall(r"File\d{1,2}", text)
print(result)  # → ['File1', 'File22', 'File33']
```

### Common Special Sequences

| Sequence | Matches |
|----------|---------|
| `\d` | Any digit (0–9) |
| `\D` | Any non-digit |
| `\w` | Any word character (letters, digits, underscore) |
| `\W` | Any non-word character |
| `\s` | Any whitespace (space, tab, newline) |
| `\S` | Any non-whitespace |
| `^` | Start of string |
| `$` | End of string |

<Callout type="info">
**Always use raw strings (`r"..."`) for regex patterns.** The `r` prefix tells Python to treat backslashes literally. Without it, `\d` might be interpreted as an escape sequence before regex even sees it. This is the single most common regex gotcha in Python.
</Callout>

## More `re` Functions

### `re.sub()` — Search and Replace

```python
text = "I love Java. Java is great."

result = re.sub("Java", "Python", text)
print(result)  # → I love Python. Python is great.
```

`sub()` replaces **all occurrences** of the pattern. You can limit replacements with the `count` parameter:

```python
result = re.sub("Java", "Python", text, count=1)
print(result)  # → I love Python. Java is great.
```

### `re.split()` — Split on a Pattern

```python
text = "apple,banana;orange:grape"

result = re.split(r"[,;:]", text)
print(result)  # → ['apple', 'banana', 'orange', 'grape']
```

This splits the string wherever it finds a comma, semicolon, or colon — far more flexible than the basic `str.split()` method.

### Practical Example: Extract All Words from Messy Text

```python
text = "Hello !@ How %#% are *# you doing @ today>"

words = re.findall(r"\w+", text)
print(words)  # → ['Hello', 'How', 'are', 'you', 'doing', 'today']
```

The `\w+` pattern matches one or more word characters, cleanly extracting words while ignoring all punctuation and special characters.

## Groups and Backreferences — Advanced Pattern Magic

**Groups** let you capture parts of a match using parentheses. This is invaluable when you need to extract specific pieces from a larger pattern.

### Capturing Groups

```python
text = "My email is sandipan@example.com"

pattern = r"(\w+)@(\w+)\.(\w+)"
match = re.search(pattern, text)

if match:
    print("Full match:", match.group(0))   # → sandipan@example.com
    print("Username:", match.group(1))     # → sandipan
    print("Domain:", match.group(2))       # → example
    print("Extension:", match.group(3))    # → com
```

Each pair of parentheses creates a numbered group:
- `group(0)` — the entire match
- `group(1)` — first set of parentheses (username)
- `group(2)` — second set (domain)
- `group(3)` — third set (extension)

### Backreferences — Reusing Captured Groups

Backreferences let you **reference captured groups** in replacement patterns. This is incredibly powerful for data transformation:

```python
text = "The date is 25-12-2024"

pattern = r"(\d{2})-(\d{2})-(\d{4})"
result = re.sub(pattern, r"\3/\2/\1", text)
print(result)  # → The date is 2024/12/25
```

Here's what happened:
- `\1` refers to group 1 (day: `25`)
- `\2` refers to group 2 (month: `12`)
- `\3` refers to group 3 (year: `2024`)

The replacement pattern `\3/\2/\1` rearranges them into year/month/day format. This technique is essential for format conversion tasks.

### Named Groups

For complex patterns, numbered groups become hard to read. Named groups solve this:

```python
pattern = r"(?P<day>\d{2})-(?P<month>\d{2})-(?P<year>\d{4})"
match = re.search(pattern, "25-12-2024")

if match:
    print(match.group("day"))    # → 25
    print(match.group("month"))  # → 12
    print(match.group("year"))   # → 2024
```

## Regex Flags — Controlling Match Behavior

Flags modify how the pattern engine operates. The most commonly used:

### `re.IGNORECASE` — Case-Insensitive Matching

```python
text = "Python is AWESOME"

result = re.findall("awesome", text, re.IGNORECASE)
print(result)  # → ['AWESOME']
```

Without the flag, this would return an empty list because the cases don't match.

### Other Useful Flags

| Flag | Effect |
|------|--------|
| `re.IGNORECASE` (or `re.I`) | Case-insensitive matching |
| `re.MULTILINE` (or `re.M`) | `^` and `$` match at line boundaries, not just string boundaries |
| `re.DOTALL` (or `re.S`) | `.` matches newline characters too |
| `re.VERBOSE` (or `re.X`) | Allows comments and whitespace in patterns for readability |

### Verbose Mode for Complex Patterns

```python
email_pattern = re.compile(r"""
    ^                   # Start of string
    [\w.+-]+            # Username (word chars, dots, plus, hyphen)
    @                   # Literal @ symbol
    [\w-]+              # Domain name
    \.                  # Literal dot
    [a-zA-Z]{2,}        # TLD (at least 2 letters)
    $                   # End of string
""", re.VERBOSE)

print(email_pattern.match("user@example.com"))   # → Match
print(email_pattern.match("invalid@.com"))        # → None
```

## Project: Password Strength Validator

Let's build a practical tool that checks password strength using everything we've learned:

```python
import re

def validate_password(password):
    """Check password strength against common security rules."""
    checks = [
        (len(password) >= 8, "Must be at least 8 characters long"),
        (re.search(r"[A-Z]", password), "Must contain at least one uppercase letter"),
        (re.search(r"[a-z]", password), "Must contain at least one lowercase letter"),
        (re.search(r"\d", password), "Must contain at least one digit"),
        (re.search(r"[!@#$%^&*(),.?\":{}|<>]", password),
         "Must contain at least one special character"),
    ]

    failures = [msg for passed, msg in checks if not passed]

    if failures:
        return {"strong": False, "issues": failures}
    return {"strong": True, "issues": []}


# Test with different passwords
test_passwords = ["weak", "NoDigits!", "short1!", "StrongPass1!", "Tr0ub4dor&3"]

for pwd in test_passwords:
    result = validate_password(pwd)
    status = "STRONG" if result["strong"] else "WEAK"
    print(f"  '{pwd}' → {status}")
    for issue in result["issues"]:
        print(f"    - {issue}")
```

Output:

```
  'weak' → WEAK
    - Must be at least 8 characters long
    - Must contain at least one uppercase letter
    - Must contain at least one digit
    - Must contain at least one special character
  'NoDigits!' → WEAK
    - Must contain at least one digit
  'short1!' → WEAK
    - Must be at least 8 characters long
    - Must contain at least one uppercase letter
  'StrongPass1!' → STRONG
  'Tr0ub4dor&3' → STRONG
```

## Regex Cheat Sheet

```python
import re

# Search for first match
re.search(r"pattern", text)

# Find all matches
re.findall(r"pattern", text)

# Match at start only
re.match(r"pattern", text)

# Search and replace
re.sub(r"pattern", "replacement", text)

# Split on pattern
re.split(r"pattern", text)

# Compile for reuse (performance)
pattern = re.compile(r"pattern", re.IGNORECASE)
pattern.findall(text)
```

| Pattern | Matches |
|---------|---------|
| `.` | Any character (except newline) |
| `\d` | Digit (0-9) |
| `\w` | Word character (a-z, A-Z, 0-9, _) |
| `\s` | Whitespace |
| `[abc]` | Any of a, b, or c |
| `[^abc]` | Not a, b, or c |
| `a*` | Zero or more `a` |
| `a+` | One or more `a` |
| `a?` | Zero or one `a` |
| `a{3}` | Exactly 3 `a` |
| `a{2,5}` | 2 to 5 `a` |
| `^` | Start of string |
| `$` | End of string |
| `(...)` | Capture group |
| `(?P<name>...)` | Named capture group |
