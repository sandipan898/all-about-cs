# How to Install Python & Set Up Your Development Environment

Before you write your first line of Python, you need two things in place: the **Python interpreter** and a comfortable editor to work in. Getting this right from the start saves hours of frustration later. This guide walks you through every step — on both Linux and Windows — so you can go from a blank machine to running Python code in under 15 minutes.

<YouTubeEmbed id="2OCWHaCHDzg" title="How to Install Python & Set Up Your Development Environment (Linux & Windows)" />

## Key Takeaways

- Python often comes pre-installed on Linux, but it may default to Python 2 — here's how to fix that.
- Windows users install Python via an official `.exe` installer from `python.org`.
- **Visual Studio Code (VS Code)** is the recommended editor: free, lightweight, and packed with Python tooling.
- Two VS Code extensions — **Python** (by Microsoft) and **AREPL** — dramatically improve your coding experience as a beginner.
- You can switch between multiple Python environments directly inside VS Code.

## Part 1 — Installing Python

### On Linux

Most Linux distributions ship with Python already installed. Open a terminal and check what version you have:

```bash
python --version
# or, if that points to Python 2:
python3 --version
```

If the output shows **Python 2.x**, or if Python 3 isn't installed at all, install Python 3.8 with:

```bash
sudo apt-get update
sudo apt-get install python3.8
```

Once installed, verify it:

```bash
python3 --version
# Expected output: Python 3.8.x
```

<Callout type="info">
**Python 2 vs Python 3 on Linux:** Many Linux systems still ship with Python 2 as the default `python` binary. Throughout this course — and in professional Python development — always use `python3` and `pip3` explicitly until you've configured Python 3 as your system default. This one habit prevents a surprising number of compatibility errors.
</Callout>

A quick reference for the two-command world:

| Intent | Python 2 default system | Python 3 default system |
|---|---|---|
| Run a script | `python3 script.py` | `python script.py` |
| Install a package | `pip3 install pkg` | `pip install pkg` |
| Check version | `python3 --version` | `python --version` |

### On Windows

Windows does not bundle Python. You need to download and install the official interpreter.

1. Go to [python.org/downloads](https://www.python.org/downloads/) and click the **Download Python 3.x.x** button. The site auto-detects your OS.
2. Run the downloaded `.exe` file.
3. **Critical step:** On the first screen of the installer, check the box that says **"Add Python to PATH"** before clicking Install Now.

> 🖼️ **Visual Suggestion:** Screenshot of the Python Windows installer with the "Add Python to PATH" checkbox highlighted.

<Callout type="warning">
**Don't skip "Add to PATH."** Without it, your terminal won't find the `python` command and you'll need to reinstall. It's the single most common stumbling block for new Windows users.
</Callout>

After installation, open **Command Prompt** (`Win + R` → type `cmd`) and verify:

```bash
python --version
# Expected output: Python 3.x.x

pip --version
# Expected output: pip xx.x from C:\Users\...\Python3x\lib\site-packages\pip (python 3.x)
```

Both commands should respond cleanly. If they do, Python is ready on Windows.

## Part 2 — Installing Visual Studio Code

**VS Code** is a free, open-source editor from Microsoft. It supports virtually every programming language, but its Python tooling is especially mature — making it the go-to choice for beginners and professionals alike.

### On Linux

You have two installation paths:

**Option A — Via the Snap Store (recommended for beginners):**

```bash
sudo snap install --classic code
```

**Option B — Via the `.deb` package:**

Download the `.deb` installer from [code.visualstudio.com](https://code.visualstudio.com/), then install it:

```bash
sudo dpkg -i code_*.deb
sudo apt-get install -f   # resolves any missing dependencies
```

### On Windows

Download the Windows installer from [code.visualstudio.com](https://code.visualstudio.com/), run the `.exe`, and follow the setup wizard. Accept the defaults — they're sensible.

## Part 3 — Configuring VS Code for Python

### Step 1: Install the Python Extension

Open VS Code. Create a new file and save it with a `.py` extension — for example, `hello_world.py`. VS Code will immediately detect that you're working with Python and show a prompt in the bottom-right corner recommending the official Python extension.

If the prompt doesn't appear:

1. Click the **Extensions** icon in the left sidebar (or press `Ctrl+Shift+X`).
2. Search for `Python`.
3. Install the extension published by **Microsoft** (it's the first result).

This single extension unlocks:

- **IntelliSense** — smart autocompletion as you type
- **Linting** — real-time error and style checking
- **Code snippets** — shortcuts that expand into common patterns
- **Integrated debugging** — set breakpoints and step through code
- **Environment switching** — seamlessly move between Python versions or virtual environments

### Step 2: Select Your Python Interpreter

After installing the extension, look at the **bottom-left corner** of the VS Code window. You'll see something like `Python 3.8.5`. Clicking it opens a picker where you can choose from any Python installation or virtual environment on your machine.

> 🖼️ **Visual Suggestion:** Screenshot of the VS Code status bar with the Python interpreter selector highlighted at the bottom-left.

<Callout type="info">
**Why does interpreter selection matter?** If you later create virtual environments for different projects (a common professional practice), this picker lets you switch between them instantly — without touching a terminal.
</Callout>

### Step 3: Write and Run "Hello, World!"

With the extension installed and interpreter selected, write your first Python program:

```python
# hello_world.py

print("Hello, World!")
```

Run it by clicking the **▶ Run** button that appears at the top-right corner of the editor. The integrated terminal opens at the bottom and prints:

```
Hello, World!
```

Alternatively, you can run it from the terminal directly:

```bash
# Linux / macOS
python3 hello_world.py

# Windows
python hello_world.py
```

Both routes produce the same result. The run button is simply a convenience wrapper around that terminal command.

## Part 4 — The AREPL Extension: Real-Time Code Evaluation

Once you've confirmed your environment works, there's one more extension worth adding: **AREPL for Python**.

Install it the same way as before — open the Extensions panel (`Ctrl+Shift+X`), search `AREPL`, and install the first result.

### What AREPL Does

AREPL evaluates your Python code **as you type**, showing output in a live preview panel beside your file. You don't need to save and run — you just write, and the results appear instantly.

```python
# Try this in a file with AREPL active:

numbers = [1, 2, 3, 4, 5]
squared = [n ** 2 for n in numbers]
print(squared)
# AREPL shows: [1, 4, 9, 16, 25] — immediately, as you finish typing
```

Activate it by clicking the **orange AREPL button** in the top-right corner of any open `.py` file. A split panel appears on the right side, updating live with every keystroke.

> 🖼️ **Visual Suggestion:** Side-by-side screenshot of a VS Code window — Python code on the left, AREPL live output panel on the right.

<Callout type="info">
**AREPL is optional but powerful.** For beginners learning basic syntax and expressions, the instant feedback loop removes the friction of the save-run-check cycle. It's particularly useful when experimenting with list comprehensions, string formatting, or small algorithmic ideas.
</Callout>
